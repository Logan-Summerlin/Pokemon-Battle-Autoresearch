## 1. Critical Fixes (Will Crash on First Run)

### Fix 1: Missing Python Dependencies in `pyproject.toml`

**Problem:** The forked repo's `pyproject.toml` only lists 4 dependencies:
```toml
dependencies = [
    "torch>=2.4.0,<2.7",
    "numpy>=1.26.0",
    "lz4>=4.3.0",
    "huggingface_hub>=0.20.0",
]
```

`scripts/train_phase4.py` imports `tqdm` (progress bars), `rich` (formatted logging), and likely `pyyaml`. The training script will crash immediately on `import tqdm` or `import rich`.

**Fix:** Update `pyproject.toml` dependencies to:
```toml
dependencies = [
    "torch>=2.4.0,<2.7",
    "numpy>=1.26.0",
    "lz4>=4.3.0",
    "huggingface_hub>=0.20.0",
    "tqdm>=4.66.0",
    "rich>=13.7.0",
    "pyyaml>=6.0",
]
```

Optional but recommended (add if `wandb` tracking is desired):
```toml
    "wandb>=0.16.0,<0.18",
```

### Fix 2: Missing `data/processed/vocabs/gen3ou/` Subdirectory

**Problem:** The source repository has BOTH root-level vocab files AND a `gen3ou/` subdirectory with Gen 3-specific vocabularies:
```
data/processed/vocabs/
├── abilities.json        # root-level (generic)
├── actions.json
├── items.json
├── moves.json
├── species.json
├── status.json
├── terrain.json
├── types.json
├── weather.json
└── gen3ou/               # <-- MISSING from forked repo
    ├── abilities.json    # Gen 3-specific vocab
    ├── actions.json
    ├── items.json
    ├── moves.json
    ├── species.json
    ├── status.json
    ├── terrain.json
    ├── types.json
    └── weather.json
```

The forked repo only has the 9 root-level files. If `train_phase4.py` or the tensorizer loads vocabs from the `gen3ou/` subdirectory path, training will fail with a `FileNotFoundError`.

**Fix:** Copy all 9 vocab files from the source repo's `data/processed/vocabs/gen3ou/` into the forked repo at the same path. Alternatively, verify which path `train_phase4.py` actually loads from — if it uses only root-level vocabs, this is not blocking.

**Verification command:**
```bash
grep -r "gen3ou" scripts/train_phase4.py src/data/tensorizer.py src/data/dataset.py
```

### Fix 3: Missing `data/processed/priors.json`

**Problem:** `src/data/priors.py` loads metagame priors from `data/processed/priors.json`. If the training pipeline calls `load_priors()`, this file must exist.

**Fix:** Copy `priors.json` from the source repo, or verify it gets regenerated by `process_dataset.py`.

## 2. Medium-Priority Fixes (Functional but Suboptimal)

### Fix 4: Missing Output Directories

**Problem:** Several directories referenced by the harness scripts don't exist:
- `Autoresearch/results/` — `eval_harness.py --output` writes here
- `experiments/` — referenced in the RunPod guide for per-experiment outputs

**Fix:** Add `.gitkeep` files:
```bash
mkdir -p Autoresearch/results && touch Autoresearch/results/.gitkeep
mkdir -p experiments && touch experiments/.gitkeep
```

The `eval_harness.py` does call `output_path.parent.mkdir(parents=True, exist_ok=True)` so it will create `results/` on first run, but having it in the repo makes the structure explicit.

### Fix 5: Missing `data/processed/metadata.json`

**Problem:** This file contains dataset statistics (100K perspectives, 75.9K battles, Elo distribution). It's generated by `process_dataset.py` and not needed at training time, but useful for agents to understand the dataset.

**Fix:** Either copy from source repo or document that it's generated during `process_dataset.py`.

### Fix 6: No Experiment Config Template

**Problem:** Only `Autoresearch/configs/anchor.yaml` exists. Agents creating new experiments don't have a template showing the expected YAML format.

**Fix:** Add `Autoresearch/configs/template.yaml` with all configurable fields and comments:
```yaml

#

# Unchanged values inherit from the anchor (AR-000).

# Architecture

ffn_multiplier: 3       # 2-4 range

# Embeddings
species_embedding_dim: 48
move_embedding_dim: 24
item_embedding_dim: 16
ability_embedding_dim: 16
type_embedding_dim: 12

# Training
max_window: 2           # KEY VARIABLE: 2-20 range

epochs: 30
lr: 0.0001
weight_decay: 0.01
warmup_steps: 300
grad_accum: 1
num_battles: 50000      # Up to 100000 available

# Loss
aux_weight: 0.2
use_value_head: false
value_weight: 0.1

# Efficiency
prune_dead_features: true
amp: auto               # off, fp16, bf16, auto
torch_compile: false

# Reproducibility
seed: 42
```

## 4. Permission Configuration Assessment

### Current State

**Codex (`.codex/config.toml`):**
```toml
writable_roots = ["Autoresearch/run_experiment.py"]
network_access = false
```
Correctly locked to single file. No issues.

**Claude Code (`.claude/settings.json`):**
Allows editing: `Autoresearch/**`, `configs/**`, `scripts/train_phase4.py`, `src/models/battle_transformer.py`, `src/data/dataset.py`, `src/data/auxiliary_labels.py`

Denies editing: `observation.py`, `tensorizer.py`, `replay_parser.py`, `data/splits/**`, tests.

### Recommendation

The current Claude Code permissions are appropriate for the AUTORESEARCH_IMPLEMENTATION_PLAN's "approved edit surface." However, if the goal is to restrict ALL agents to only `run_experiment.py`:

Replace `.claude/settings.json` allow list with:
```json
"Edit(Autoresearch/run_experiment.py)",
"Write(Autoresearch/results/**)",
"Write(Autoresearch/notes/**)"
```

And add to deny:
```json
"Edit(src/**)",
"Edit(scripts/**)"
```

This creates a "pure experiment loop" where agents can only configure and launch experiments, not modify the underlying code.
